# Azure Firewall (Azure Firewall)

ကောင်းမွန်စွာ ဒီဇိုင်းထုတ်ထားသော firewall သည် သင်၏ဒေတာနှင့် resource များအတွက် အရေးကြီးသော လုံခြုံရေး အတားအဆီးတစ်ခုအဖြစ် လုပ်ဆောင်သည်။ ခွင့်ပြုချက်မရှိဘဲ ဝင်ရောက်ခြင်းမှ တန်ဖိုးရှိသော ပိုင်ဆိုင်မှုများကို ကာကွယ်ရန် ဝင်လာသောနှင့် ထွက်သွားသော ဒေတာကို စစ်ဆေးခြင်း။ ယခုအချိန်တွင် firewall များသည် ဤအရေးကြီးသော လုပ်ငန်းကို မည်သို့လုပ်ဆောင်သည်ကို သင်အတော်အတန် လေ့လာပြီးပြီဖြစ်သော်လည်း သင့်ကွန်ရက်မှတဆင့် ယာဉ်အသွားအလာ စီးဆင်းမှုကို စီမံခန့်ခွဲရန် စည်းမျဉ်းများ ဖန်တီးခြင်းထက် ပိုမိုများပြားပါသည်။ ၎င်းသည် scalability၊ မြင့်မားသော ရရှိနိုင်မှုနှင့် အခြားအချက်များကဲ့သို့သော အချက်များကိုလည်း ထည့်သွင်းစဉ်းစားပြီး သင်၏ Azure ပတ်ဝန်းကျင်အတွက် ပြည့်စုံသော လုံခြုံရေးကို သေချာစေသည်။ နောက်မိနစ်အနည်းငယ်အတွင်း ဖွဲ့စည်းပုံအတွက် အကောင်းဆုံး အလေ့အကျင့်များ၊ လုံခြုံသော ဗဟို firewall အကောင်အထည်ဖော်ခြင်းနှင့် ဆက်စပ်နေသော အင်္ဂါရပ်များနှင့် firewall များသည် segmentation ကို ပံ့ပိုးပေးသော ခိုင်မာသော လုံခြုံရေး အနေအထားကို မည်သို့ အထောက်အကူပြုနိုင်သည်ကို စူးစမ်းလေ့လာခြင်းဖြင့် firewall များအကြောင်း သင်၏နားလည်မှုကို နက်ရှိုင်းစေမည်ဖြစ်သည်။ ပထမဦးစွာ Azure Firewall ၏ အကျဉ်းချုပ်ကို လေ့လာကြည့်ကြပါစို့။ Azure Firewall သည် သင်၏ Azure virtual network ကို ကာကွယ်ရန် ဗဟိုမှ စီမံခန့်ခွဲပြီး ဒီဇိုင်းထုတ်ထားသော Cloud အခြေပြု ဝန်ဆောင်မှုတစ်ခုဖြစ်သည်။ ဤအခြေအနေတွင် စီမံခန့်ခွဲခြင်းဆိုသည်မှာ Cloud ပံ့ပိုးပေးသူအနေဖြင့် Azure သည် firewall ၏ ထိရောက်မှုကို သေချာစေရန်၊ ၎င်း၏ နေ့စဉ်လုပ်ငန်းဆောင်ရွက်မှုများနှင့် ပြုပြင်ထိန်းသိမ်းမှုတို့ကို ကြီးကြပ်ရန် တာဝန်ရှိသည်ဟု ဆိုလိုသည်။ သင်တွင် မည်သည့်တာဝန်များရှိပြီး ဤဝန်ဆောင်မှုအပေါ် မည်သည့်ထိန်းချုပ်မှုများ ပြုလုပ်နိုင်သနည်း။ ကောင်းပြီ၊ firewall ဖွဲ့စည်းပုံတွင် အကောင်းဆုံး အလေ့အကျင့်အချို့ကို အကောင်အထည်ဖော်ခြင်းဖြင့် လုံခြုံရေးကို သိသိသာသာ မြှင့်တင်နိုင်သည်။ ဥပမာအားဖြင့် Azure firewall ကို မည်သည့်ကွန်ရက်တွင်မဆို အသုံးချနိုင်သည်မှာ မှန်သော်လည်း အကောင်းဆုံး အလေ့အကျင့်များသည် ဗဟိုကွန်ရက်တစ်ခု ဖန်တီးပြီး ဤမူလတစ်ခုမှတဆင့် နောက်ထပ်ကွန်ရက်များအားလုံးကို လမ်းကြောင်းပြရန် အကြံပြုထားသည်။ Firewall များသည် application တစ်ခုသို့ ဝင်လာသောနှင့် ထွက်သွားသော ယာဉ်အသွားအလာကို ထိန်းညှိခြင်းထက် ကျော်လွန်ပြီး segmentation ကို အတင်းအကြပ် လုပ်ဆောင်ရန်အတွက်လည်း အသုံးပြုပြီး တစ်ဦးချင်း virtual network တစ်ခုစီတွင် အသုံးချနိုင်သည်။ ၎င်းသည် application များသို့ ဝင်လာသောနှင့် ထွက်သွားသော ယာဉ်အသွားအလာကိုသာမက resource များအကြားတွင်လည်း ကန့်သတ်ထားကြောင်း သေချာစေပါသည်။ ၎င်းကို ဧရိယာ သို့မဟုတ် application တစ်ခုသို့ ဝင်ရောက်ခွင့်ပြုထားသော IP address များ၏ bandwidth ကို ကန့်သတ်ခြင်းဖြင့် ရရှိသည်။ ဥပမာအားဖြင့် firewall ကို လက်ခံဆောင်ရွက်ရန် သီးသန့်ထားရှိသော ကွန်ရက်တစ်ခုသည် ယာဉ်အသွားအလာကို စစ်ထုတ်ရန်အတွက် ဗဟို gateway တစ်ခုအဖြစ် လုပ်ဆောင်နိုင်သည်။ ကွန်ရက်တစ်ခုစီအတွက် firewall တစ်ခု တည်ဆောက်မည့်အစား ယာဉ်အသွားအလာအားလုံးသည် ဗဟို gateway မှတဆင့် ပထမဦးစွာ ဖြတ်သန်းသွားသည်။ ထို့အပြင် ၎င်းတွင် Cloud အခြေပြု resource များ သို့မဟုတ် အင်တာနက်နှင့် ယေဘုယျအားဖြင့် မထိတွေ့မီ on-premises ယာဉ်အသွားအလာများ ပါဝင်နိုင်သည်။ ဤနည်းဖြင့် သင့်ကွန်ရက်ကို ဖွဲ့စည်းခြင်းဖြင့် သင်သည် အဓိက အားသာချက်များစွာကို ရရှိနိုင်သည်၊ သို့သော် ၎င်းသည် အားလုံးမဟုတ်ပါ။ Azure Firewall ၏ စွမ်းရည်များကို ထပ်မံမြှင့်တင်ပေးသော အဓိက အင်္ဂါရပ်များစွာ ရှိပါသည်။ Cloud အခြေပြု resource များစွာကဲ့သို့ပင် Azure Firewall သည် ယာဉ်အသွားအလာ ဝယ်လိုအားအပေါ် အခြေခံ၍ scaling တက်ခြင်း၏ မူလအားသာချက်ကို ပေးပါသည်။ scalability ၏ အားသာချက်များသည် သင်၏လုပ်ငန်း သဘောသဘာဝအပေါ် များစွာမူတည်ပါသည်။ ဥပမာအားဖြင့် ရာသီအလိုက် ကုန်သွယ်မှုပြုလုပ်သော သို့မဟုတ် ယာဉ်အသွားအလာ အထွတ်အထိပ်များ ကြုံတွေ့ရသော အဖွဲ့အစည်းတစ်ခုသည် တည်ငြိမ်သော၊ ပို၍တိုင်းတာနိုင်သော ဝင်ရောက်မှုပုံစံရှိသော အဖွဲ့အစည်းတစ်ခုထက် scalability ကို ပိုမိုအလေးပေးပါလိမ့်မည်။ Microsoft သည် အမြဲတမ်း ကွန်ရက်ရရှိနိုင်မှုကို သေချာစေရန် တာဝန်ယူသည်။ ၎င်းကိုရရှိရန်အတွက် မတူညီသော ပထဝီဝင်တည်နေရာများတွင် redundant instances များကို ထိန်းသိမ်းထားသည့် failover လုံခြုံရေး အစီအမံများကို အကောင်အထည်ဖော်ထားသည်။ Failover ဆိုသည်မှာ သင်၏ ဝန်ဆောင်မှု ပေးပို့မှုကို ထိခိုက်စေသော ပြဿနာတစ်ခု ပေါ်ပေါက်ပါက Microsoft သည် backup တစ်ခုက အားလျော့သွားကြောင်း သေချာစေပါသည်။ ဖွဲ့စည်းနိုင်မှုနှင့် စစ်ထုတ်ခြင်းတို့ဖြင့် သင်သည် firewall ကို စိတ်ကြိုက်ပြင်ဆင်နိုင်ပြီး IP address များ၊ port များ၊ protocol များနှင့် fully qualified domain names သို့မဟုတ် FQDNs ဟုသိကြသည့် အမျိုးမျိုးသော စံနှုန်းများအပေါ် အခြေခံ၍ စစ်ထုတ်ခြင်းကို အသုံးချနိုင်သည်။ ဤ granular အဆင့် စစ်ထုတ်ခြင်းသည် တိကျသော ယာဉ်အသွားအလာ စီမံခန့်ခွဲမှုကို ခွင့်ပြုပေးပြီး ဆိုလိုသည်မှာ လူမှုမီဒီယာ ဆိုက်များကဲ့သို့သော ယေဘုယျ domain အမည်များကို သင်ဖယ်ထုတ်နိုင်သည်ဟု ဆိုလိုသည်။ စီမံခန့်ခွဲသူများသည် သီးခြား port များ သို့မဟုတ် protocol များကို ရွေးချယ်ခြင်းဖြင့် ပြင်ပ ခြိမ်းခြောက်မှု သရုပ်ဆောင်များကို ပိုမိုထိရောက်စွာ ထိန်းချုပ်နိုင်သည်။ Source နှင့် destination network address translation စွမ်းရည်များ သို့မဟုတ် SNAT နှင့် DNAT သည် အတွင်းပိုင်းကွန်ရက်နှင့် ပြင်ပကမ္ဘာအကြား ဆက်သွယ်မှုကို လွယ်ကူချောမွေ့စေသည်။ SNAT သည် virtual network မှ ထွက်သွားသော ယာဉ်အသွားအလာသည် public IP address တစ်ခုမှ စတင်သည်ဟု ထင်ရအောင် သေချာစေပြီး DNAT သည် virtual network အတွင်းရှိ သင့်လျော်သော private IP address သို့ ဝင်လာသော အင်တာနက် ယာဉ်အသွားအလာကို ဘာသာပြန်ပေးသည်။ ဤဘာသာပြန်ခြင်းနှင့် စစ်ထုတ်ခြင်း လုပ်ငန်းစဉ်သည် virtual network နှင့် ကျယ်ပြန့်သော အင်တာနက်အကြား ချောမွေ့ပြီး လုံခြုံသော ဆက်သွယ်မှုကို လွယ်ကူချောမွေ့စေသည်။ destination တစ်ခုတွင် ပုံမှန်အားဖြင့် IP address တစ်ခုသာရှိသော်လည်း Azure Firewall သည် destination တစ်ခုနှင့် ဆက်စပ်ရန် IP address များစွာကို ခွင့်ပြုသည်။ Microsoft Threat Intelligence သည် ဆိုက်ဘာတိုက်ခိုက်မှုများနှင့် ဟက်ကင်းလုပ်ဆောင်မှုများ၏ သက်သေအထောက်အထားများကို စုဆောင်းသည့် ဝန်ဆောင်မှုတစ်ခုဖြစ်သည်။ ၎င်းသည် load balancing၊ ယာဉ်အသွားအလာ segmentation နှင့် ထွက်သွားသော ဆက်သွယ်ရေးအတွက် အထောက်အကူဖြစ်နိုင်သည်။ ဤဒေတာကို Microsoft intelligence အဖွဲ့မှ လုပ်ဆောင်နိုင်သော ထိုးထွင်းသိမြင်မှုများ ဖန်တီးရန် လုပ်ဆောင်သည်။ ဤထိုးထွင်းသိမြင်မှုများကို Defender နှင့် Sentinel ကဲ့သို့သော Microsoft ဝန်ဆောင်မှုအမျိုးမျိုးတွင် ပေါင်းစပ်ထည့်သွင်းထားပြီး ဤစနစ်များသည် သီးခြားအဖွဲ့အစည်းတစ်ခုကို တိုက်ရိုက်ပစ်မှတ်မထားသည့်တိုင် ကျယ်ပြန့်သော တိုက်ခိုက်မှုများမှ သင်ယူနိုင်စေပါသည်။ ဤစုပေါင်းသင်ယူမှုသည် Microsoft ဝန်ဆောင်မှုများကို အသုံးပြုသော အဖွဲ့အစည်းများ၏ လုံခြုံရေးကို မြှင့်တင်ပေးပါသည်။ နောက်ဆုံးအနေဖြင့် Azure Monitor နှင့် ပေါင်းစပ်ခြင်းသည် ရရှိသော ထိုးထွင်းသိမြင်မှုများမှတဆင့် အနာဂတ် အားနည်းချက်များကို ပိတ်ဆို့ခြင်းနှင့် ကာကွယ်ရေး အစီအမံတစ်ခုအဖြစ် ယာဉ်အသွားအလာကို ခွဲခြမ်းစိတ်ဖြာခြင်းနှင့် စောင့်ကြည့်ခြင်းတို့ကို အထောက်အကူပြုပါသည်။ Azure Firewall ၏ ဤအဓိက အင်္ဂါရပ်များသည် ၎င်း၏ စွမ်းရည်များကို ထပ်မံမြှင့်တင်ပေးပါသည်။ ဗဟိုမှ စီမံခန့်ခွဲထားသော firewall ၏ အရေးပါမှုကို နားလည်ခြင်းနှင့် ဗဟိုကွန်ရက်တစ်ခုတွင် အသုံးချခြင်းနှင့် segmentation ကို အသုံးချခြင်းကဲ့သို့သော အဓိက အလေ့အကျင့်များကို လက်ခံကျင့်သုံးခြင်းဖြင့် သင်သည် သင့်ကွန်ရက်ကို ပေါင်းစည်းထားသော၊ ချောမွေ့သော လုံခြုံရေးချဉ်းကပ်မှုတစ်ခုဖြင့် စွမ်းဆောင်နိုင်ပါသည်။
