# Cloud security posture management (CSPM) in depth (Cloud လုံခြုံရေး အနေအထား စီမံခန့်ခွဲမှု (CSPM) အသေးစိတ်)

## Introduction (နိဒါန်း)

Cloud လုံခြုံရေး အနေအထား စီမံခန့်ခွဲမှု (CSPM) သည် အဖွဲ့အစည်းတစ်ခုတွင် ခိုင်မာသော လုံခြုံရေး စီမံခန့်ခွဲမှုကို အားကောင်းစေရန်နှင့် အတင်းအကျပ် အကောင်အထည်ဖော်ရန်အတွက် ရရှိနိုင်သော အလေ့အကျင့်များနှင့် ကိရိယာများအတွက် အသုံးအနှုန်းဖြစ်သည်။ ဆိုက်ဘာလုံခြုံရေး ပညာရှင်တစ်ဦးသည် ကုမ္ပဏီပိုင်ဆိုင်မှုများကို ကာကွယ်ရန်အတွက် လျှော့ချရေး မဟာဗျူဟာများစွာကို အသုံးပြုနိုင်သည်။

ဤစာဖတ်ခြင်းသည် CSPM အကောင်အထည်ဖော်ခြင်းဆိုင်ရာ ဆွေးနွေးမှုကို ဆက်လက်လုပ်ဆောင်ပြီး အန္တရာယ်ရှာဖွေခြင်း၊ အရိပ် IT လျှော့ချခြင်းနှင့် လုံခြုံရေးချို့ယွင်းမှုများကို ရှောင်ရှားရာတွင် နည်းပညာမူဝါဒ၏ အခန်းကဏ္ဍကဲ့သို့သော နောက်ထပ် ထည့်သွင်းစဉ်းစားမှုများကို ချဲ့ထွင်ပါသည်။

## Discover risks (အန္တရာယ်များကို ရှာဖွေပါ)

IT ဆိုက်ဘာ ကျွမ်းကျင်သူတစ်ဦး၏ အခန်းကဏ္ဍတွင် ကုမ္ပဏီ၏ ဉာဏပစ္စည်းကို အန္တရာယ်ပြုခံရခြင်း ရှိမရှိကို သိရှိခြင်း ပါဝင်သည်။ ၎င်းကို မူပိုင်ခွင့်ချိုးဖောက်မှုအတွက် YouTube ဗီဒီယိုများကို အလိုအလျောက် စကင်ဖတ်ခြင်း သို့မဟုတ် ဒေတာချိုးဖောက်မှုများအတွက် အမှောင် ဝဘ်ကို စကင်ဖတ်ခြင်းကဲ့သို့သော ဧရိယာများစွာတွင် တွေ့ရှိနိုင်သည်။

ဤအခန်းကဏ္ဍ၏ ရှုထောင့်တစ်ခုတွင် တားမြစ်ထားသော စကားဝှက်စာရင်းများ ဖန်တီးခြင်း ပါဝင်သည်။ ထို့အပြင် IT ဌာနမှ အတည်ပြုခြင်း သို့မဟုတ် အတည်ပြုခြင်းမရှိသော စက်ပစ္စည်းများ သို့မဟုတ် အပလီကေးရှင်းများကို အသုံးပြုခြင်းကို ရည်ညွှန်းသည့် အရိပ် IT ၏ စောင့်ကြည့်ခြင်းလည်း ပါဝင်သည်။

ဥပမာအားဖြင့်၊ ကုဒ်ရေးသူတစ်ဦးသည် အလားအလာရှိသော အမှားများ၊ ကုဒ်ရေးခြင်းပုံစံ ချိုးဖောက်မှုများနှင့် အခြားပြဿနာများကို အလိုအလျောက် စစ်ဆေးခြင်းဖြင့် ကုဒ်အရည်အသွေးကို မြှင့်တင်ရန် ဒီဇိုင်းထုတ်ထားသော ကိရိယာဖြစ်သည့် အများသုံး linting ဝန်ဆောင်မှုကို အသုံးပြုသည့် အခြေအနေကို စဉ်းစားကြည့်ကြပါစို့။ ဤအခြေအနေတွင် ကုမ္ပဏီ၏ ကုဒ်ကို စေ့စေ့စပ်စပ် ခွဲခြမ်းစိတ်ဖြာခြင်းနှင့် အကောင်းဆုံးဖြစ်အောင် ပြုလုပ်ရန် ပြင်ပဝန်ဆောင်မှုတစ်ခုသို့ တင်သွင်းပါသည်။ အခြားဥပမာတစ်ခုမှာ developer များသည် အများသုံး GitHub repository ကို အသုံးပြု၍ ကုမ္ပဏီကုဒ်တွင် ပူးပေါင်းလုပ်ဆောင်သည့်အခါ ဖြစ်ပေါ်လာသည်။

အန္တရာယ်များကို ရှာဖွေခြင်းတွင် သတင်းအချက်အလက် မသိသော သို့မဟုတ် ဂရုမစိုက်သော ဝန်ထမ်းတစ်ဦး၏ လုပ်ဆောင်ချက်များကြောင့် ဖြစ်ပေါ်လာနိုင်သည့် ရည်ရွယ်ချက်ရှိသောနှင့် မရည်ရွယ်သော ခြိမ်းခြောက်မှုများကို ခွဲခြားသတ်မှတ်ခြင်း ပါဝင်သည်။

## Technical policy (နည်းပညာမူဝါဒ)

နည်းပညာမူဝါဒသည် ကုမ္ပဏီမှ ပြဋ္ဌာန်းထားသော စံနှုန်းများနှင့် အကောင်းဆုံး အလေ့အကျင့်များနှင့် သက်ဆိုင်ပါသည်။ ကုမ္ပဏီဒေတာကို ကိုင်တွယ်ရန် မသင့်လျော်သော ခွင့်ပြုချက်မရှိသော စက်ပစ္စည်းများနှင့် အပလီကေးရှင်းများကို အသုံးပြုခြင်း ပါဝင်သည့် အရိပ် IT ကဲ့သို့သော ပြဿနာများကို ဖြေရှင်းရာတွင် အရေးကြီးသော အခန်းကဏ္ဍမှ ပါဝင်ပါသည်။

ဝန်ထမ်းများက မည်သည့်အရာကို ခွင့်ပြုနိုင်သည်ကို သိရှိရန်အတွက် ကုမ္ပဏီ၏ အကောင်းဆုံး အလေ့အကျင့်များနှင့် စံနှုန်းများကို အကြမ်းဖျင်းဖော်ပြသည့် ဗဟိုပြုစာရွက်စာတမ်းတစ်ခု လိုအပ်ပါသည်။ ၎င်းတွင် ဝန်ထမ်းတစ်ဦးအား အချို့သော အပြုအမူများကို လုပ်ဆောင်ခြင်းမှ တားဆီးပေးသည့် ကာကွယ်ရေးအစီအမံများ အကောင်အထည်ဖော်ခြင်း ပါဝင်သည်။ ဥပမာအားဖြင့်၊ အထိခိုက်မခံသော အချက်အလက်များကို ကူးယူခြင်း သို့မဟုတ် ဖျက်ခြင်းမှ တားဆီးရန် မူဝါဒတစ်ခုကို ပြဋ္ဌာန်းနိုင်သည်။ ၎င်းတွင် ဝန်ထမ်းစက်ပစ္စည်းများကို စကင်ဖတ်ခြင်းနှင့် စနစ်အတွက် ကျန်းမာရေးမကောင်းဟု ယူဆသော စက်ပစ္စည်းများသို့ ဝင်ရောက်ခွင့်ကို တားဆီးခြင်းတို့လည်း ပါဝင်နိုင်သည်။

ဤအယူအဆ၏ အရေးပါသော အစိတ်အပိုင်းတစ်ခုမှာ ကုမ္ပဏီစံနှုန်းများ၊ ဝန်ထမ်းလုပ်ဆောင်ချက်များ၊ ဆိုက်ဘာလုံခြုံရေး၏ လက်ရှိအခြေအနေနှင့် ကုမ္ပဏီအချက်အလက်ကို ယနေ့အထိ ကာကွယ်ထားသည့် ဒီဂရီတို့ကို အဆက်မပြတ် စစ်ဆေးခြင်းနှင့် ပြန်လည်သုံးသပ်ခြင်း၏ အလေ့အကျင့်ဖြစ်သည်။

## Minimizing risks: A holistic security posture (အန္တရာယ်များကို လျှော့ချခြင်း- အလုံးစုံ လုံခြုံရေး အနေအထား)

ခိုင်မာသော လုံခြုံရေး အနေအထား၏ အခြေခံမူများကို နားလည်ရာတွင် အဓိက ရည်မှန်းချက်မှာ စနစ်သို့ အန္တရာယ်များကို လျှော့ချရန် ဖြစ်သည်။ ၎င်းတွင် ချိုးဖောက်မှုတစ်ခုကြောင့် ဖြစ်ပေါ်နိုင်သည့် အန္တရာယ်ကို လျှော့ချခြင်းနှင့် ခွင့်ပြုချက်မရှိဘဲ ဝင်ရောက်ရန် ကြိုးပမ်းသည့် ဟက်ကာတစ်ဦးအတွက် ရင်းနှီးမြှုပ်နှံမှုအပေါ် ပြန်အမ်းငွေကို လျှော့ချခြင်းတို့ ပါဝင်သည်။

ကုဒ်ဝှက်ခြင်းဆိုင်ရာ သင်ခန်းစာကို စဉ်းစားပါ။ စကားဝှက်၏ hash ၏ ကုဒ်ဝှက်ထားသော ဗားရှင်းကို သိမ်းဆည်းခြင်းဖြင့် စနစ်ကို ချိုးဖောက်သည့် မည်သည့် ဟက်ကာမဆို စကားဝှက်များကို ကုဒ်ဖြည်၍မရပါက ၎င်းတို့ပြုလုပ်သည့် ကြိုးပမ်းအားထုတ်မှုအပေါ် အနည်းဆုံး ပြန်အမ်းငွေကိုသာ ရရှိမည်ဖြစ်သည်။ ၎င်းသည် ချိုးဖောက်ရန် ကြိုးပမ်းမှုမှန်သမျှကို တားဆီးရန်အတွက် အတားအဆီးတစ်ခုအဖြစ် လုပ်ဆောင်နိုင်သည်။ ပိုမို အထိခိုက်မခံသော အချက်အလက်များအတွက် ဒေတာကို ပိုမို ကုဒ်ဝှက်နိုင်သည့် Azure ဝန်ဆောင်မှုများအကြောင်း သင်လေ့လာခဲ့ပြီးဖြစ်သည်။ လွှဲပြောင်းနေစဉ်တွင် ၎င်းကို ကုဒ်ဝှက်ပြီး operating system မှ ၎င်းကို လုပ်ဆောင်သည့်အတိုင်း ဒေတာကိုပင် ကုဒ်ဝှက်ပါ။

## CSPM in different departments (မတူညီသော ဌာနများတွင် CSPM)

Cloud လုံခြုံရေး အနေအထား စီမံခန့်ခွဲမှုသည် ဆိုက်ဘာလုံခြုံရေး ဌာနတွင်သာ ကန့်သတ်မထားကြောင်း ထိုက်တန်စွာ မှတ်သားသင့်ပါသည်။ ၎င်းသည် အဖွဲ့အစည်းအတွင်း တစ်ကမ္ဘာလုံးက လိုက်နာရမည့် အလုံးစုံ ချဉ်းကပ်မှုတစ်ခု ဖြစ်သင့်သည်။

- ခြိမ်းခြောက်မှု ထောက်လှမ်းရေး အဖွဲ့- ခြိမ်းခြောက်မှု ထောက်လှမ်းရေး အဖွဲ့သည် cloud ပတ်ဝန်းကျင်၏ လုံခြုံရေး အနေအထားကို အဆက်မပြတ် အကဲဖြတ်ရန်နှင့် တိုက်ခိုက်သူများက အသုံးချနိုင်သည့် ဖြစ်နိုင်ချေရှိသော မှားယွင်းသော ဖွဲ့စည်းမှုများ သို့မဟုတ် အားနည်းချက်များကို ခွဲခြားသတ်မှတ်ရန် CSPM ကို အသုံးပြုသည်။
- သတင်းအချက်အလက် နည်းပညာ (IT): IT ဌာနသည် cloud အရင်းအမြစ်များကို လုံခြုံရေး မူဝါဒများနှင့် မှန်ကန်စွာ ဖွဲ့စည်းထားပြီး လိုက်နာကြောင်း သေချာစေရန် လုံခြုံရေး စစ်ဆေးမှုများကို အလိုအလျောက် လုပ်ဆောင်ရန် CSPM ကို အသုံးပြုသည်။
- လိုက်နာမှုနှင့် အန္တရာယ် စီမံခန့်ခွဲမှု အဖွဲ့များ- လိုက်နာမှုနှင့် အန္တရာယ် စီမံခန့်ခွဲမှု အဖွဲ့များသည် စည်းမျဉ်းလိုအပ်ချက်များကို လိုက်နာကြောင်း သေချာစေရန်နှင့် ဒေတာချိုးဖောက်မှု အန္တရာယ်ကို လျှော့ချရန် cloud လုံခြုံရေး ထိန်းချုပ်မှုများကို စောင့်ကြည့်ပြီး အတင်းအကျပ် အကောင်အထည်ဖော်ရန် CSPM ကို အသုံးပြုသည်။
- စီးပွားရေး ခေါင်းဆောင်များနှင့် SME များ (ဘာသာရပ်ဆိုင်ရာ ကျွမ်းကျင်သူများ)- စီးပွားရေး ခေါင်းဆောင်များနှင့် SME များသည် ၎င်းတို့၏ cloud ပိုင်ဆိုင်မှုများ၏ လုံခြုံရေး အခြေအနေကို မြင်နိုင်စွမ်းရရှိရန်၊ အသိပေးဆုံးဖြတ်ချက်ချခြင်းနှင့် အန္တရာယ် လျှော့ချခြင်းတို့ကို လုပ်ဆောင်နိုင်စေရန် CSPM ကို အသုံးပြုသည်။
- လုံခြုံရေး ဗိသုကာနှင့် လုပ်ငန်းဆောင်ရွက်မှုများ- လုံခြုံရေး ဗိသုကာနှင့် လုပ်ငန်းဆောင်ရွက်မှု အဖွဲ့သည် ဖြစ်နိုင်ချေရှိသော ခြိမ်းခြောက်မှုများအတွက် တက်ကြွသော တုံ့ပြန်မှုများကို လုပ်ဆောင်နိုင်စေရန်နှင့် ကောင်းမွန်စွာ ကာကွယ်ထားသော cloud ပတ်ဝန်းကျင်ကို သေချာစေရန် real-time တွင် လုံခြုံရေး ပြဿနာများကို ရှာဖွေရန် CSPM ကို အသုံးချသည်။
- စစ်ဆေးရေး အဖွဲ့- စစ်ဆေးရေး အဖွဲ့သည် cloud အရင်းအမြစ်များ၏ လုံခြုံရေး အနေအထားကို အကဲဖြတ်ရန်နှင့် လုံခြုံရေး မူဝါဒများနှင့် အကောင်းဆုံး အလေ့အကျင့်များကို လိုက်နာမှုကို ခြေရာခံရန် အကဲဖြတ် ကိရိယာတစ်ခုအဖြစ် CSPM ကို အသုံးပြုသည်။

## Security tools (လုံခြုံရေး ကိရိယာများ)

ယခု လုံခြုံရေးကို သေချာစေရန် အသုံးပြုသည့် ကိရိယာများကို စူးစမ်းလေ့လာကြည့်ကြပါစို့။

- Microsoft Defender: အုပ်ချုပ်ရေး အင်္ဂါရပ်များ၊ စည်းမျဉ်း လိုက်နာမှု အတည်ပြုခြင်းနှင့် cloud လုံခြုံရေး ရှာဖွေသူတို့ကို ပံ့ပိုးပေးသည်။ ခွင့်ပြုချက်ကို စီမံခန့်ခွဲပြီး တိုက်ခိုက်မှု ခွဲခြမ်းစိတ်ဖြာမှုကို ပံ့ပိုးပေးသည်။
- Azure လုံခြုံရေး စင်တာ: ဝန်ထမ်း စောင့်ကြည့်ခြင်း၊ ခြိမ်းခြောက်မှု အကဲဖြတ်ခြင်းနှင့် လုံခြုံရေး အကြံပြုချက်များ
- Microsoft Sentinel: လုံခြုံရေး ဒေတာကို စုဆောင်းပြီး ဖြစ်နိုင်ချေရှိသော တိုက်ခိုက်မှု ပုံစံများကို ခွဲခြားသတ်မှတ်ရန် ၎င်းတို့ကို စုစည်းကာ ချိုးဖောက်ပြီးနောက်ပိုင်းတွင် မှုခင်းဆေးပညာကို ပံ့ပိုးပေးသည်။
- Azure Monitor: အခြား ဝန်ဆောင်မှုများမှ တယ်လီမီတာ ဒေတာကို တင်ပြပြီး ပြသပေးသည်။ သင့် cloud ပတ်ဝန်းကျင်တွင် ဖြစ်ပေါ်နေသော လုပ်ဆောင်ချက်များသို့ အပိုထိုးထွင်းသိမြင်မှုကို ခွင့်ပြုပေးသည်။
- Azure Bastion: ဟက်ကာများအတွက် ရရှိနိုင်သည့် တိုက်ခိုက်မှု မျက်နှာပြင်ကို လျှော့ချရန်အတွက် သင့် SSH (Secure Shell) နှင့် RDP (Remote Desktop Protocol) ports များကို လုံခြုံရေး buffer တစ်ခုနောက်တွင် ထုပ်ပိုးပေးသည်။

## Conclusion (နိဂုံး)

CSPM သည် နည်းပညာ ကျွမ်းကျင်သူများမှ စီးပွားရေး ခေါင်းဆောင်များအထိ အဖွဲ့အစည်းတစ်ခု၏ ရှုထောင့်တိုင်း ပါဝင်သည့် ပြောင်းလဲနေသောနှင့် ပူးပေါင်းလုပ်ဆောင်သော ချဉ်းကပ်မှုတစ်ခုဖြစ်သည်။ ဤအလေ့အကျင့်များနှင့် ကိရိယာများကို ပေါင်းစပ်ခြင်းဖြင့် အဖွဲ့အစည်းများသည် ၎င်းတို့၏ လုံခြုံရေး အနေအထားကို အားကောင်းစေပြီး ပေါ်ပေါက်လာသော ခြိမ်းခြောက်မှုများကို ထိရောက်စွာ လျှော့ချနိုင်သည်။

ရှေ့ဆက်မည့် ခရီးတွင် ဤစာဖတ်ခြင်းတွင် မီးမောင်းထိုးပြထားသည့် ကိရိယာများနှင့် အလေ့အကျင့်များကို ပိုမိုနက်ရှိုင်းစွာ စူးစမ်းလေ့လာရန် အခွင့်အရေး ရရှိပါမည်။
