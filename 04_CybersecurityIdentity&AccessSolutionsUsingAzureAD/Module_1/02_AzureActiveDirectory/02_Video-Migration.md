# ပြောင်းရွှေ့ခြင်း

ကလောက်အခြေပြု Active Directory ၏ အကျိုးကျေးဇူးများကို သုတေသနပြုပြီးနောက်၊ ဆမ်သည် သူမ၏ လျင်မြန်စွာ တိုးတက်နေသော ကုမ္ပဏီ၏ အကျိုးစီးပွားအတွက် ရုပ်ပိုင်းဆိုင်ရာ Active Directory မှ Azure AD သို့ ပြောင်းရွှေ့ရန် ဆုံးဖြတ်ခဲ့သည်။

သို့သော်၊ ၎င်းကို ဖြစ်ပေါ်စေရန် သူမဘာလုပ်ရမည်ကို အတိအကျ မသေချာသေးပါ။ အသုံးပြုသူအကောင့် ဒေတာများကို လွှဲပြောင်းရန် လိုအပ်သော အချိန်ကြောင့် ထုတ်လုပ်မှု ဆုံးရှုံးမည်ကို သူမ စိုးရိမ်နေသည်။

ကံကောင်းစွာဖြင့်၊ ဆမ်မကြာမီ ရှာဖွေတွေ့ရှိမည်ဖြစ်သည့်အတိုင်း၊ စေ့စပ်သေချာသော စီစဉ်မှုနှင့် မှန်ကန်သော ကိရိယာများဖြင့် ဤလုပ်ငန်းစဉ်သည် သူမထင်ထားသည်ထက် ပိုမိုလွယ်ကူသည်။

ဤဗီဒီယိုတွင်၊ အသုံးပြုသူများနှင့် စက်ပစ္စည်းများကို မည်သို့တင်သွင်းသည်၊ အသုံးပြုသော ကိရိယာအချို့၏ အကြမ်းဖျင်း၊ နှင့် AAD ကို အသုံးပြုသောအခါ အဆင့်မြင့် အင်္ဂါရပ်အချို့ကို သင်သွားမည်ဖြစ်သည်။

ယခု ပြောင်းရွှေ့ခြင်းလုပ်ငန်းစဉ်၏ အဆင့်လေးဆင့်ကို ဖွင့်ကြည့်ကြပါစို့၊ ၎င်းကို ပြောင်းရွှေ့မှုမတိုင်ခင် အကဲဖြတ်ခြင်းဖြင့် စတင်သည်။

ထို့ကြောင့် ရှိပြီးသား AD အခြေခံအဆောက်အအုံကို အကဲဖြတ်ပါ၊ ပြောင်းရွှေ့ရန် လိုအပ်သော အရာဝတ္ထုများကို ဖော်ထုတ်ပါ၊ နှင့် ရုပ်ပိုင်းဆိုင်ရာ AD နှင့် အခြားတက်ကြွသော ဝန်ဆောင်မှုများပေါ်တွင် မှီခိုမှုများကို ဆုံးဖြတ်ပါ။ ၎င်းသည် AD မှ လက်ရှိစီမံခန့်ခွဲနေသော အရာဝတ္ထုများ၏ အသေးစိတ် စာရင်းကို လုပ်ဆောင်ရန် အခွင့်အရေးတစ်ခုဖြစ်သည်။

ဤအရာဝတ္ထုများတွင် အသုံးပြုသူများ၊ အုပ်စုများ၊ နှင့် စက်ပစ္စည်းများ ပါဝင်သည်။ အဖွဲ့အစည်းတစ်ခုတွင် ပံ့ပိုးပေးသော အဆင့်များနှင့် စာရွက်စာတမ်းများနှင့်အတူ အသုံးပြုထားသော SaaS အမျိုးမျိုး ရှိမည်ဖြစ်သည်။

အထူးသဖြင့်၊ AD ကို အထောက်အထားစစ်ဆေးခြင်းနှင့် ခွင့်ပြုချက်အတွက် မှီခိုနေသော စိတ်ကြိုက်အပ္ပလီကေးရှင်းများ သို့မဟုတ် ဝန်ဆောင်မှုများကို ထည့်သွင်းစဉ်းစားပါ။ ဤအပ္ပလီကေးရှင်းများတွင် ၎င်းတို့၏ ကိုယ်ပိုင်ပြင်ဆင်သတ်မှတ်မှုများ ရှိသောကြောင့် ပြဿနာတစ်ခု ဖြစ်ပေါ်လာပါက၊ ဖြေရှင်းချက်သည် ကျယ်ပြန့်သော ပြဿနာဖြေရှင်းမှု အားထုတ်မှုများ လိုအပ်သည်။ အကြောင်းမှာ စိတ်ကြိုက်အပ္ပလီကေးရှင်းများသည် အွန်လိုင်းအသိုင်းအဝိုင်းသို့ လွတ်လပ်စွာ ပြဿနာဖြေရှင်းရန် ထိတွေ့မှုကို ကန့်သတ်ထားသောကြောင့်ဖြစ်သည်။

ဒုတိယအဆင့်မှာ ရုပ်ပိုင်းဆိုင်ရာ AD နှင့် Azure Active Directory အကြား ချိတ်ဆက်မှုတစ်ခု ထူထောင်ခြင်းဖြစ်သည်။ ၎င်းကို Azure AD Connect ကို အသုံးပြု၍ လုပ်ဆောင်နိုင်ပြီး၊ ၎င်းသည် ရုပ်ပိုင်းဆိုင်ရာ AD မှ အသုံးပြုသူအကောင့်များနှင့် စကားဝှက်များကို Azure AD သို့ ထပ်တူပြုစေသည်။

Azure AD Connect သည် ရုပ်ပိုင်းဆိုင်ရာ AD တွင် ပြုလုပ်သော ပြောင်းလဲမှုများကို Azure AD တွင် ထင်ဟပ်စေရန် သေချာစေသည်။ Azure AD Connect သည် ရုပ်ပိုင်းဆိုင်ရာ AD ဆာဗာပေါ်တွင် ဆော့ဖ်ဝဲလ်ကို ထည့်သွင်းခြင်းဖြင့် အလုပ်လုပ်သည်။

ထို့နောက် ၎င်းသည် AD မှ စီမံခန့်ခွဲသော အသုံးပြုသူများ၊ အုပ်စုများ၊ နှင့် စက်ပစ္စည်းများအကြောင်း အချက်အလက်များကို စုဆောင်းပြီး ဤဒေတာကို AAD သို့ ပေးပို့သည်။ ထို့အပြင်၊ ၎င်းသည် AAD တွင် ပြုလုပ်သော ပြောင်းလဲမှုများကို လက်ခံရရှိပြီး ရုပ်ပိုင်းဆိုင်ရာ ဆက်တင်များနှင့် ထပ်တူပြုသည်။

AAD Connect သည် AAD ဖန်တီးပြီးနောက်တွင်လည်း ၎င်းကို ဆက်လက်လုပ်ဆောင်သည်။ အဓိကအားဖြင့်၊ ၎င်းသည် ဆက်တင်နှစ်ခုလုံးကို ညှိထားစေသော တံတားတစ်ခုအဖြစ် လုပ်ဆောင်သည်။

ယခု AAD ကို ဖန်တီးပြီး AD နှင့် ညှိထားပြီးဖြစ်သောကြောင့်၊ တတိယအဆင့်မှာ AD မှ အသုံးပြုသူအကောင့်များနှင့် အုပ်စုများကို AAD သို့ ပြောင်းရွှေ့ခြင်းဖြစ်သည်။ Microsoft သည် ၎င်းကို အောင်မြင်စေမည့် ကိရိယာများစွာကို ပံ့ပိုးပေးသည်၊ ဥပမာ Azure AD Connect ကိရိယာကဲ့သို့ဖြစ်သည်။

ဤကိရိယာသည် စီမံခန့်ခွဲသူများအား ပြောင်းရွှေ့ရန် လိုအပ်သော အသုံးပြုသူများနှင့် အုပ်စုများကို ရွေးချယ်ပြီး ၎င်းတို့ကို AAD သို့ လွှဲပြောင်းနိုင်စေသည်။ ထို့အပြင် လိုအပ်သော အရည်အချင်းများနှင့် ဂုဏ်သတ္တိများအားလုံးကို မှန်ကန်စွာ လွှဲပြောင်းပေးကြောင်း သေချာစေသည်။

အသုံးပြုသူအုပ်စုများ ဖန်တီးခြင်းသည် AAD ၏ အဆင့်မြင့် အင်္ဂါရပ်တစ်ခုဖြစ်သည်။ သင့်တွင် ရရှိနိုင်သော လိုင်စင်ပေါ်မူတည်၍ အသုံးပြုမှု နှင့် အတိုင်းအတာ ကျယ်ပြန့်မှုကို သတိပြုပါ။

အသုံးပြုသူများနှင့် အုပ်စုများကို ပြောင်းရွှေ့ရန် အသုံးပြုနိုင်သော နောက်ထပ်ကိရိယာတစ်ခုမှာ Active Directory Migration Tool (ADMT) ဖြစ်သည်။ ADMT သည် စကားဝှက်များ၊ အုပ်စုအဖွဲ့ဝင်မှုများ၊ နှင့် အသုံးပြုသူများနှင့် အုပ်စုများနှင့် ဆက်စပ်နေသော အခြားအရည်အချင်းများကို ထုတ်ယူနိုင်ပြီး ၎င်းတို့ကို ကလောက်တွင် ပုံတူပွားနိုင်သည်။

၎င်းသည် AD နှင့် မှတ်ပုံတင်ထားသော ကွန်ပျူတာများနှင့် အကြောင်းအရာများကိုလည်း ပြောင်းရွှေ့နိုင်သည်။

ဤလုပ်ငန်းစဉ်၏ နောက်ဆုံးအဆင့်မှာ AD ပေါ်တွင် မှီခိုနေသော အပ္ပလီကေးရှင်းများနှင့် ဝန်ဆောင်မှုများကို AAD သို့ ပြောင်းရွှေ့ခြင်းဖြစ်သည်။ ၎င်းတွင် အပ္ပလီကေးရှင်းများနှင့် ဝန်ဆောင်မှုများကို AD အစား AAD ကို အထောက်အထားစစ်ဆေးခြင်းနှင့် ခွင့်ပြုချက်အတွက် အသုံးပြုရန် ပြန်လည်ပြင်ဆင်သတ်မှတ်ခြင်း ပါဝင်သည်။

စက်ပစ္စည်းများနှင့် ဝန်ဆောင်မှုများတွင် ကွန်ရက်သို့ ချိတ်ဆက်ပုံနှင့် ၎င်းတို့ ဝင်ရောက်နိုင်သော အချက်အလက်များနှင့် သက်ဆိုင်သော အထူးပြင်ဆင်သတ်မှတ်မှုများ မကြာခဏရှိသည်။ ဤနေရာတွင် ADMT သည် ဤ AAD လုံခြုံရေး အင်္ဂါရပ်များကို အကောင်အထည်ဖော်နိုင်သည်။

ဤဗီဒီယိုတွင်၊ သင်သည် ရုပ်ပိုင်းဆိုင်ရာ AD မှ Azure AD သို့ ပြောင်းရွှေ့ခြင်း လုပ်ငန်းစဉ်ကို ဖွင့်ထုတ်ခဲ့သည်။ ယခု သင်သည် ပါဝင်သော အဓိကအဆင့်များကို သိရှိသင့်ပြီဖြစ်သည်။ ၎င်းတို့မှာ ပြောင်းရွှေ့မှုမတိုင်ခင် အကဲဖြတ်ခြင်း ပြုလုပ်ခြင်း၊ ရုပ်ပိုင်းဆိုင်ရာ AD ကို Azure AD နှင့် ထပ်တူပြုခြင်း၊ အသုံးပြုသူများနှင့် အုပ်စုများကို ပြောင်းရွှေ့ခြင်း၊ နှင့် အပ္ပလီကေးရှင်းများနှင့် ဝန်ဆောင်မှုများကို ပြောင်းရွှေ့ခြင်းတို့ ဖြစ်သည်။

သင်သည် Microsoft မှ လမ်းတစ်လျှောက် ကူညီပေးနိုင်သော ကိရိယာများစွာကိုလည်း မိတ်ဆက်ခံခဲ့ရသည်။ ၎င်းတို့တွင် AD နှင့် AAD အကြား အကောင့်များနှင့် စကားဝှက်များကို ထပ်တူပြုပေးသော Azure AD Connect ပါဝင်သည်။

ထို့အပြင်၊ အသုံးပြုသူနှင့် အုပ်စု အရည်အချင်းများကို လွယ်ကူစွာ ပုံတူပွားပြီး ပြောင်းရွှေ့ခွင့်ပြုသော Active Directory Migration Tool လည်း ပါဝင်သည်။

ယခု ဆမ်သည် ပြောင်းရွှေ့ခြင်းလုပ်ငန်းစဉ်မှ ဘာကို မျှော်လင့်ရမည်ကို သိရှိပြီးဖြစ်သောကြောင့်၊ အချိန်ဆုံးရှုံးမှု သို့မဟုတ် ထုတ်လုပ်မှု ဆုံးရှုံးမည်ကို စိုးရိမ်စရာမလိုဘဲ ယုံကြည်စိတ်ချစွာ စီစဉ်နိုင်သည်။
