# Exercise: Create a Windows Virtual Machine in the Azure Portal (လေ့ကျင့်ခန်း: Azure Portal တွင် Windows ဗဟိုကွန်ပျူတာတစ်လုံး ဖန်တီးခြင်း)

## Case Study (ဖြစ်ရပ်လေ့လာမှု)

ဤလေ့ကျင့်ခန်းတွင် သင်သည် Azure တွင် Windows ဗဟိုကွန်ပျူတာတစ်လုံးကို ဖန်တီးမည်ဖြစ်ပြီး၊ Sam’s Scoops အတွက် ဝက်ဘ်ဆိုက်အသစ်တစ်ခုကို လက်ခံဆောင်ရွက်ရန် အသုံးပြုမည်ဖြစ်သည်။ သင်သည် Sam’s Scoops ဝက်ဘ်ဆာဗာအတွက် ဗဟိုကွန်ပျူတာကို 172.16.1.0 ကွန်ရက်ခွဲ (subnet) ပေါ်ရှိ "Web_Server" ဗဟိုကွန်ရက်ကို အသုံးပြု၍ ဖြန့်ကျက်မည်ဖြစ်သည်။ ဤဗဟိုကွန်ပျူတာ၏ အမည်ကို "SamScoopsWeb" ဟု သတ်မှတ်ရမည်ဖြစ်သည်။

ဤလေ့ကျင့်ခန်းသည် သင့်အား အဆင့်များမှတစ်ဆင့် လမ်းညွှန်ပေးမည်ဖြစ်သည်။

### Azure User Interface (Azure အသုံးပြုသူ အင်တာဖေ့စ်)

Microsoft Cloud ကိရိယာများ၏ တက်ကြွသော သဘောသဘာဝကြောင့်၊ ဤသင်တန်းအကြောင်းအရာ ဖန်တီးပြီးနောက် Azure အသုံးပြုသူ အင်တာဖေ့စ် (UI) တွင် ပြောင်းလဲမှုများ ဖြစ်ပေါ်နိုင်သည်။ ထို့ကြောင့် ဗီဒီယိုများ၊ စာဖတ်ခြင်းများ သို့မဟုတ် လေ့ကျင့်ခန်းများရှိ မျက်နှာပြင်ဓာတ်ပုံများသည် သင်တွေ့ကြုံရသော UI နှင့် အတိအကျ မကိုက်ညီနိုင်ပါ။ သို့သော် ဤပြောင်းလဲမှုများသည် UI ၏ လုပ်ဆောင်ချက်များကို ထိခိုက်မှုမရှိကြောင်း သတိပြုပါ။ ထို့ကြောင့် ဗီဒီယို၊ စာဖတ်ခြင်း သို့မဟုတ် လေ့ကျင့်ခန်းတွင် ပြထားသော အဆင့်အားလုံးကို သင်ဆက်လက် လုပ်ဆောင်နိုင်မည်ဖြစ်သည်။

## Create the Network and Resource Group (ကွန်ရက်နှင့် အရင်းအမြစ်အုပ်စု ဖန်တီးခြင်း)

### အဆင့် ၁ (Step 1)

သင့် အထောက်အထားများဖြင့် **Azure Portal** သို့ ဝင်ရောက်ပါ။

### အဆင့် ၂ (Step 2)

Azure Portal မီနူးတွင်၊ မျက်နှာပြင်၏ ဘယ်ဘက်ခြမ်းတွင် တည်ရှိသော **"အရင်းအမြစ်ဖန်တီးရန်" (Create a Resource)** ခလုတ်ကို ရွေးပါ။

### အဆင့် ၃ (Step 3)

ရှာဖွေဘားတွင် **"Virtual Network"** ဟု ရိုက်ထည့်ပါ၊ ရလဒ်များမှ **"Virtual Network"** ကို ရွေးပြီး **"ဖန်တီးရန်" (Create)** ကို ရွေးချယ်ပါ။

### အဆင့် ၄ (Step 4)

**"ဗဟိုကွန်ရက်ဖန်တီးရန် လမ်းညွှန်" (Create a Virtual Network Wizard)** ၏ "အခြေခံအချက်အလက်များ" (Basics) တက်ဘ်တွင် အောက်ပါအချက်အလက်များကို ဖြည့်စွက်ပါ-

- **စာရင်းသွင်းမှု (Subscription):** သင်အသုံးပြုလိုသော စာရင်းသွင်းမှုကို ရွေးပါ။
- **အရင်းအမြစ်အုပ်စု (Resource Group):** "အသစ်ဖန်တီးရန်" (Create New) ကို ရွေးပြီး အရင်းအမြစ်အုပ်စုအသစ်၏ အမည်အဖြစ် **"RG_Web_Server"** ကို ထည့်သွင်းပါ။
- **အမည် (Name):** ဗဟိုကွန်ရက်၏ အမည်အဖြစ် **"Web_Server"** ကို ထည့်သွင်းပါ။
- **ဒေသ (Region):** သင့်အနီးဆုံးဖြစ်သော ဒေသကို ရွေးပါ။

### အဆင့် ၅ (Step 5)

**"IP လိပ်စာများ" (IP Addresses)** ကို ရွေးပါ။

- မူလသတ်မှတ်ထားသော လိပ်စာနေရာအား **"ကွန်ရက်ခွဲထည့်ရန်" (Add a Subnet)** ဘေးရှိ အစက်သုံးစက် (...) ကို ရွေးပြီး **"လိပ်စာနေရာ ဖျက်ရန်" (Delete Address Space)** ကို ရွေးချယ်ခြင်းဖြင့် ဖျက်ပါ�।
- သတိပေးချက်တစ်ခု ပေါ်လာမည်: _"သင်သည် ဗဟိုကွန်ရက်သို့ အနည်းဆုံး လိပ်စာနေရာတစ်ခု ထည့်ရမည်။"_
- **"လိပ်စာနေရာ ထည့်ရန်" (Add an Address Space)** ကို ရွေးပါ�。
- စတင်လိပ်စာ (Starting Address): **172.16.1.0** ကို ဖြည့်ပါ။
- လိပ်စာနေရာ အရွယ်အစား (Address Space Size): **/24 (256 Addresses)** ကို ဖြည့်ပါ။
- **"ထည့်ရန်" (Add)** ကို ရွေးပါ။

### အဆင့် ၆ (Step 6)

ယခု ကွန်ရက်ခွဲ (Subnet) တစ်ခု ထည့်ရမည်ဖြစ်သည်။

- **"+ ကွန်ရက်ခွဲ ထည့်ရန်" (+ Add a Subnet)** ကို ရွေးပါ။
- အသေးစိတ်အချက်အလက်များကို မူလသတ်မှတ်ထားသည့်အတိုင်း ထားပါ။
- **"ထည့်ရန်" (Add)** ကို ရွေးပါ။

### အဆင့် ၇ (Step 7)

**"ပြန်လည်သုံးသပ်ပြီး ဖန်တီးရန်" (Review + Create)** ခလုတ်ကို ရွေးပြီး ဆက်တင်များကို ပြန်လည်သုံးသပ်ပါ။

### အဆင့် ၈ (Step 8)

**"ဖန်တီးရန်" (Create)** ခလုတ်ကို ရွေးပြီး ဗဟိုကွန်ရက်ကို ဖန်တီးပါ။

- ဤအဆင့်တွင် "ဖြန့်ကျက်မှု ဆောင်ရွက်နေသည်" (Deployment in Progress) ဟူသော အကြောင်းကြားချက်တစ်ခု ပေါ်လာမည်ဖြစ်သည်။
- ဖြန့်ကျက်မှု ပြီးစီးသွားသောအခါ "သင့်ဖြန့်ကျက်မှု ပြီးစီးပါပြီ" (Your Deployment is Complete) ဟု ပြသမည်ဖြစ်သည်။

ဗဟိုကွန်ရက် ဖန်တီးပြီးသည်နှင့်၊ Sam’s Scoops ဝက်ဘ်ဆာဗာအတွက် ဗဟိုကွန်ပျူတာ ဖန်တီးမှုကို ဆက်လက်လုပ်ဆောင်နိုင်ပါသည်။

## Create the Virtual Machine (ဗဟိုကွန်ပျူတာ ဖန်တီးခြင်း)

### အဆင့် ၁ (Step 1)

သင့် အထောက်အထားများဖြင့် **Azure Portal** သို့ ဝင်ရောက်ပါ။

### အဆင့် ၂ (Step 2)

Azure Portal မီနူးတွင်၊ မျက်နှာပြင်၏ ဘယ်ဘက်ခြမ်းတွင် တည်ရှိသော **"အရင်းအမြစ်ဖန်တီးရန်" (Create a Resource)** ခလုတ်ကို ရွေးပါ။

### အဆင့် ၃ (Step 3)

ရှာဖွေဘားတွင် **"Windows Server"** ဟု ရိုက်ထည့်ပါ၊ ရလဒ်များမှ **"Windows Server"** ကို ရွေးပါ။

- **"ဖန်တီးရန်" (Create)** ကို ရွေးပြီး **"Windows Server 2022 Datacenter: Azure Edition"** ကို ရွေးချယ်ပါ။

### အဆင့် ၄ (Step 4)

**"ဗဟိုကွန်ပျူတာ ဖန်တီးရန် လမ်းညွှန်" (Create a Virtual Machine Wizard)** ၏ "အခြေခံအချက်အလက်များ" (Basics) တက်ဘ်တွင် အောက်ပါအချက်အလက်များကို ဖြည့်စွက်ပါ-

- **စာရင်းသွင်းမှု (Subscription):** သင်အသုံးပြုလိုသော စာရင်းသွင်းမှုကို ရွေးပါ။
- **အရင်းအမြစ်အုပ်စု (Resource Group):** ရှိပြီးသား အရင်းအမြစ်အုပ်စု **"RG_Web_Server"** ကို ရွေးပါ။
- **ဗဟိုကွန်ပျူတာ အမည် (Virtual Machine Name):** **"SamScoopsWeb"** ကို ထည့်သွင်းပါ။
- **ဒေသ (Region):** သင့်အနီးဆုံးဖြစ်သော ဒေသကို ရွေးပါ။
- **ပုံရိပ် (Image):** **"Windows Server 2022 Datacenter – x64 Gen2"** ကို ရွေးပါ။
- **အရွယ်အစား (Size):** သင့်ဗဟိုကွန်ပျူတာအတွက် သင့်လျော်သော အရွယ်အစားကို ရွေးပါ။ ဤလေ့ကျင့်ခန်းအတွက် အနည်းဆုံးကုန်ကျသော ရွေးချယ်မှုကို အသုံးပြုခြင်းသည် အကောင်းဆုံးဖြစ်သည်။
- **အသုံးပြုသူအမည် (Username):** **"AzAdmin"** ကို ထည့်သွင်းပါ။
- **စကားဝှက် (Password):** **"P@$$@1234567"** ကို ထည့်သွင်းပါ။
- **စကားဝှက် အတည်ပြုခြင်း (Confirm Password):** **"P@$$@1234567"** ကို ထည့်သွင်းပါ။

### အဆင့် ၅ (Step 5)

**"နောက်: ဒစ်များ" (Next: Disks)** ခလုတ်ကို ရွေးပြီး "ဒစ်များ" (Disks) တက်ဘ်သို့ ဆက်သွားပါ။

### အဆင့် ၆ (Step 6)

OS ဒစ် (OS Disk) အတွက် မူလသတ်မှတ်ထားသော ဆက်တင်များကို ထားပါ။ **"နောက်: ကွန်ရက်ချိတ်ဆက်မှု" (Next: Networking)** ခလုတ်ကို ရွေးပြီး "ကွန်ရက်ချိတ်ဆက်မှု" (Networking) တက်ဘ်သို့ ဆက်သွားပါ။

### အဆင့် ၇ (Step 7)

"ကွန်ရက်ချိတ်ဆက်မှု" (Networking) တက်ဘ်တွင် အောက်ပါ ဆက်တင်များကို ရွေးပါ-

- **ဗဟိုကွန်ရက် (Virtual Network):** **"Web_Server"** ကို ရွေးပါ။
- **ကွန်ရက်ခွဲ (Subnet):** **"default"** ကို ရွေးပါ။
- **အများသူငှာ IP (Public IP):** "အသစ်ဖန်တီးရန်" (Create New) ကို ရွေးပြီး အများသူငှာ IP လိပ်စာအသစ်အတွက် အမည်တစ်ခု ထည့်သွင်းပါ။
- **NIC ကွန်ရက် လုံခြုံရေး အုပ်စု (NIC Network Security Group):** မူလသတ်မှတ်ထားသော ဆက်တင်ကို ထားပါ။

### အဆင့် ၈ (Step 8)

**"နောက်: စီမံခန့်ခွဲမှု" (Next: Management)** ခလုတ်ကို ရွေးပြီး "စီမံခန့်ခွဲမှု" (Management) တက်ဘ်သို့ ဆက်သွားပါ။

### အဆင့် ၉ (Step 9)

"စီမံခန့်ခွဲမှု" (Management) တက်ဘ်တွင် မူလသတ်မှတ်ထားသော ဆက်တင်များကို ထားပါ။ **"နောက်: စောင့်ကြည့်မှု" (Next: Monitoring)** ခလုတ်ကို ရွေးပြီး "စောင့်ကြည့်မှု" (Monitoring) တက်ဘ်သို့ ဆက်သွားပါ။ အားလုံးကို မူလသတ်မှတ်ထားသည့်အတိုင်း ထားပါ။ ထို့နောက် **"နောက်: အဆင့်မြင့်" (Next: Advanced)** ခလုတ်ကို ရွေးပြီး "အဆင့်မြင့်" (Advanced) တက်ဘ်သို့ ဆက်သွားပါ။

### အဆင့် ၁၀ (Step 10)

"အဆင့်မြင့်" (Advanced) တက်ဘ်တွင် မူလသတ်မှတ်ထားသော ဆက်တင်များကို ထားပါ။ **"ပြန်လည်သုံးသပ်ပြီး ဖန်တီးရန်" (Review + Create)** ခလုတ်ကို ရွေးပါ။

### အဆင့် ၁၁ (Step 11)

သင့်ဗဟိုကွန်ပျူတာ၏ ဆက်တင်များကို ပြန်လည်သုံးသပ်ပြီး **"ဖန်တီးရန်" (Create)** ခလုတ်ကို ရွေးပြီး ဗဟိုကွန်ပျူတာ၏ ဖြန့်ကျက်မှုကို စတင်ပါ။ ဖြန့်ကျက်မှု ပြီးစီးသည်အထိ စောင့်ပါ။

### အဆင့် ၁၂ (Step 12)

ဖြန့်ကျက်မှု ပြီးစီးသွားသည်အထိ စောင့်ပါ။

အောင်မြင်ပါသည်။ ဗဟိုကွန်ပျူတာကို ယခု ဖြန့်ကျက်ပြီးစီးပါပြီ။ သင်သည် ဗဟိုကွန်ပျူတာသို့ ချိတ်ဆက်ရန် လိုအပ်သော နောက်အဆင့်အတွက် အဆင်သင့်ဖြစ်ပါပြီ။

---

## Connect to Virtual Machine (ဗဟိုကွန်ပျူတာသို့ ချိတ်ဆက်ခြင်း)

### အဆင့် ၁ (Step 1)

ဖြန့်ကျက်မှု ပြီးစီးသွားသည်နှင့်၊ **"အရင်းအမြစ်သို့ သွားရန်" (Go to Resource)** ခလုတ်ကို ရွေးပြီး ဗဟိုကွန်ပျူတာ စာမျက်နှာသို့ သွားပါ။

### အဆင့် ၂ (Step 2)

ဗဟိုကွန်ပျူတာ စာမျက်နှာတွင်၊ **"ချိတ်ဆက်ရန်" (Connect)** ခလုတ်ကို ရွေးပြီး ရွေးချယ်မှု မီနူးမှ **"RDP"** ကို ရွေးချယ်၍ RDP ဖိုင်ကို ဒေါင်းလုဒ်လုပ်ပါ။

### အဆင့် ၃ (Step 3)

ရွေးချယ်မှု မီနူးမှ **"RDP"** ကို ရွေးပြီး **"RDP ဖိုင် ဒေါင်းလုဒ်လုပ်ရန်" (Download RDP File)** ကို ရွေးပါ။

### အဆင့် ၄ (Step 4)

ဒေါင်းလုဒ်လုပ်ထားသော RDP ဖိုင်ကို ဖွင့်ပါ။ ဗဟိုကွန်ပျူတာ ဖန်တီးစဉ်တွင် သတ်မှတ်ထားသော အထောက်အထားများ (Credentials) ကို အသုံးပြု၍ ဗဟိုကွန်ပျူတာသို့ ချိတ်ဆက်ပါ။

သင်သည် ယခု Azure Portal တွင် "SamScoopsWeb" အတွက် Windows ဗဟိုကွန်ပျူတာတစ်လုံးကို ဖန်တီးပြီးစီးသွားပါပြီ။

---

## Clean-up (သန့်ရှင်းရေး)

သင်ကိုယ်ပိုင် Azure စာရင်းသွင်းမှုကို အသုံးပြုနေပါက၊ သင်ခန်းစာတစ်ခုစီ ပြီးဆုံးပြီးနောက် တွက်ချက်မှု အရင်းအမြစ်များကို ရပ်တန့်ရန် သန့်ရှင်းရေး ညွှန်ကြားချက်များကို လိုက်နာရန် အကြံပြုထားသည်။ သင်ကိုယ်ပိုင် စာရင်းသွင်းမှုတွင် လုပ်ဆောင်နေသည့်အခါ၊ ပရောဂျက်တစ်ခု ပြီးဆုံးသွားသောအခါ သင်ဖန်တီးထားသော အရင်းအမြစ်များကို ဆက်လက် လိုအပ်သလား ဆန်းစစ်ရန် အကောင်းဆုံးဖြစ်သည်။ လည်ပတ်နေသော အရင်းအမြစ်များသည် သင့်အား ငွေကုန်ကျနိုင်သည်။ အရင်းအမြစ်များကို တစ်ခုချင်း ဖျက်နိုင်သည် သို့မဟုတ် အရင်းအမြစ်အုပ်စု တစ်ခုလုံးကို ဖျက်ခြင်းဖြင့် အရင်းအမြစ်များအားလုံးကို ဖယ်ရှားနိုင်သည်။

လေ့ကျင့်ခန်း ပြီးစီးပြီးနောက် အရင်းအမြစ်များကို ဖျက်နည်းအတွက် ညွှန်ကြားချက်များကို **"Clean up Resources"** စာဖတ်ခြင်းတွင် ကိုးကားပါ။

---

## Conclusion (နိဂုံး)

ဤအဆင့်များကို လိုက်နာခြင်းဖြင့်၊ သင်သည် Azure Portal တွင် Windows ဗဟိုကွန်ပျူတာတစ်လုံးကို ဖန်တီးပြီး၊ ဝက်ဘ်ဆာဗာတစ်ခု ထည့်သွင်းပြီး၊ မမျှော်လင့်ထားသော အခကြေးငွေများ ရှောင်ရှားရန် အရင်းအမြစ်များကို သန့်ရှင်းပြီးစီးသွားပါပြီ။
