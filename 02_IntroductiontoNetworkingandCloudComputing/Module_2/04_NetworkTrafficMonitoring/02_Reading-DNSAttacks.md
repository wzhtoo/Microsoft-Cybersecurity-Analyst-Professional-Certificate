# DNS တိုက်ခိုက်မှုများ

## Introduction နိဒါန်း

သင်ဟာ အခု DNS က လွယ်ကူစွာ ဖတ်နိုင်တဲ့ ဝက်ဘ်ဆိုက် နာမည်တွေကို IP လိပ်စာတွေအဖြစ် ပြောင်းလဲပြီး အင်တာနက်ကို သွားလာရာမှာ ပိုအဆင်ပြေစေတယ်ဆိုတာကို သိထားပါတယ်။ ဒါက ကမ္ဘာတဝှမ်းက လူတွေ အသုံးပြုတဲ့ အသုံးဝင်တဲ့ ဝန်ဆောင်မှုတစ်ခုဖြစ်ပေမယ့်၊ အဲဒီလို များများ အသုံးပြုတာကြောင့်ပဲ ဆိုက်ဘာ တိုက်ခိုက်မှုတွေအတွက် အဓိက ပစ်မှတ်တစ်ခု ဖြစ်လာပါတယ်။ တိုက်ခိုက်မှု တစ်ခုက အနှောင့်အယှက် အများအပြား ဖြစ်စေနိုင်ပြီး၊ တိုက်ခိုက်သူတစ်ယောက်က အဲဒီကနေ အမြတ်အစွန်းတောင် ရယူနိုင်ပါတယ်။ အန္တရာယ်တွေကို သိထားတာက ခြိမ်းခြောက်မှုတွေကို ဘယ်လို လျှော့ချမလဲဆိုတာကို စဉ်းစားခင်မှာ စတင်ဖို့ နေရာကောင်းတစ်ခုပါ။

ဒီစာဖတ်ခြင်းမှာ၊ သင်ဟာ DNS ရဲ့ ဖွဲ့စည်းပုံနဲ့ ဆာဗာ အမျိုးအစား လေးမျိုးကို လေ့လာမှာဖြစ်ပြီး၊ DNS အတွင်း မတူညီတဲ့ အားနည်းချက်တွေ ဘယ်မှာ ရှိသလဲဆိုတာကို ပိုမိုကောင်းမွန်စွာ နားလည်နိုင်မှာပါ။ သင့်ကိုယ်ပိုင် ကွန်ရက်ပေါ်မှာ တစ်နေ့ မတူညီတဲ့ တိုက်ခိုက်မှုတွေကို ဖော်ထုတ်နိုင်ဖို့ ပုံမှန် DNS တိုက်ခိုက်မှုတွေကိုလည်း ဆန်းစစ်မှာပါ။

## DNS Server Types DNS ဆာဗာ အမျိုးအစားများ

အစောပိုင်းမှာ ဖော်ပြခဲ့သလို၊ ကမ္ဘာတဝှမ်းက DNS တောင်းဆိုမှု အားလုံးကို ကြည့်ရှုဖို့ ဆာဗာ တစ်ခုတည်းကိုသာ အားကိုးလို့ မရပါဘူး။ ဝန်ဆောင်မှု ပေးဖို့ ဆာဗာ အများအပြား လိုအပ်ပါတယ်။ ဆာဗာ တစ်ခုတည်းက အင်တာနက်ပေါ်က ဖြစ်နိုင်တဲ့ လိပ်စာ အားလုံးကို သိမ်းဆည်းထားလို့ မရတာမို့၊ အလုပ်တာဝန်ကို သတင်းအချက်အလက် စုဆောင်းပြီး သိမ်းဆည်းမှုမှာ မတူညီတဲ့ အခန်းကဏ္ဍတွေ ပါဝင်တဲ့ ဆာဗာ အများအပြားကြား ဖြန့်ဝေထားပါတယ်။

DNS ဆာဗာ အမျိုးအစား လေးမျိုး ရှိပြီး၊ အောက်မှာ ပုံမှာ သရုပ်ပြထားတဲ့အတိုင်း၊ ဖွဲ့စည်းပုံတစ်ခု ပေးထားပါတယ်။ အမျိုးအစား တစ်ခုစီဟာ အဲဒီ ဖွဲ့စည်းပုံထဲမှာ မတူညီတဲ့ အခန်းကဏ္ဍတစ်ခု ပါဝင်ပါတယ်။ ၎င်းတို့က **ရီကာဆစ် DNS ဆာဗာများ** (Recursive DNS servers)၊ **ရုစ် ဒိုမိန်း ဆာဗာများ** (Root domain servers)၊ **ထိပ်တန်း DNS ဆာဗာများ** (Top-level DNS servers)၊ နဲ့ **အာသော်ရီတီ DNS ဆာဗာများ** (Authoritative DNS servers) တို့ပါ။ ဒီမော်ဒယ်က အဆင့်ဆင့် ဖွဲ့စည်းထားတာမို့၊ ထိပ်ကနေ စတင်ပြီး အောက်ကို ဆင်းရပါတယ်။ ဒါမှမဟုတ် အောက်က ပုံမှာလို၊ ဘယ်ဘက်ကနေ စတင်ပြီး ထိပ်ကနေ အောက်ကို ဆင်းသွားရပါတယ်။

### Recursive DNS Server ရီကာဆစ် DNS ဆာဗာ

ပထမဆုံးက **ရီကာဆစ် ဆာဗာ** (Recursive server) – ဒါမှမဟုတ် **ဖြေရှင်းသူ** (resolver) – ဖြစ်ပြီး၊ ၎င်းက များသောအားဖြင့် သင့်ကိရိယာပေါ်က DNS ဆာဗာဖြစ်ပြီး ၎င်းရဲ့ IP လိပ်စာကို လက်ဖြင့် ဒါမှမဟုတ် DHCP ကနေ သတ်မှတ်ပေးထားပါတယ်။ ၎င်းက DNS တောင်းဆိုမှု တစ်ခု လက်ခံရရှိပြီး အဲဒီနဲ့ ရင်းနှီးနေရင်၊ ရီကာဆစ် ဖြေရှင်းသူက ၎င်းရဲ့ မှတ်ဉာဏ်ကနေ IP လိပ်စာနဲ့ ချက်ချင်း တုံ့ပြန်နိုင်ပါတယ်။ အကယ်၍ ရီကာဆစ် ဖြေရှင်းသူက IP လိပ်စာနဲ့ မရင်းနှီးရင်၊ တောင်းဆိုမှုကို ရုစ် ဒိုမိန်း ဆာဗာတွေဆီ ပို့ရပါတယ်။

### Root Domain Servers ရုစ် ဒိုမိန်း ဆာဗာများ

ကမ္ဘာတဝှမ်းမှာ ရုစ် ဒိုမိန်း ဆာဗာ ၁၃ ခု ဖြန့်ကျက်ရှိပြီး၊ အားလုံးကို **အင်တာနက် သတ်မှတ်ပေးထားသော နံပါတ်များ အာဏာပိုင်** (Internet Assigned Numbers Authority - IANA) လို့ ခေါ်တဲ့ အုပ်ချုပ်ရေး အဖွဲ့က ကြည့်ရှုထားပါတယ်။ ရုစ် ဒိုမိန်း ဆာဗာတွေရဲ့ အလုပ်က မေးခွန်းတွေကို မှန်ကန်တဲ့ တည်နေရာဆီ ညွှန်ပြဖို့ပါ။ ရုစ် ဆာဗာက မကြာသေးခင်က အဲဒီ IP ကို ကိုင်တွယ်ခဲ့ရင် အဖြေကို ပေးနိုင်မှာဖြစ်ပေမယ့်၊ မဟုတ်ရင်၊ မေးခွန်းကို အဲဒီနယ်မြေကို တာဝန်ယူတဲ့ ထိပ်တန်း ဒိုမိန်း ဆာဗာတွေဆီ ပို့ပေးပါလိမ့်မယ်။

### Top-Level DNS Servers ထိပ်တန်း DNS ဆာဗာများ

**ထိပ်တန်း ဆာဗာများ** (Top-level servers - TLD) က ကြီးမားတဲ့ နယ်မြေတွေနဲ့ နိုင်ငံတွေကို ကြည့်ရှုထားပါတယ်။ .com၊ .co.uk၊ .net၊ နဲ့ .org လို ဒိုမိန်း နာမည် အဆက် (extensions) တွေဟာ အားလုံး မတူညီတဲ့ TLD ဆာဗာတွေပါ။ ဒီဆာဗာတွေကလည်း မကြာသေးခင်က ဒီတောင်းဆိုမှုကို လက်ခံရရှိခဲ့ရင် မှန်ကန်တဲ့ တည်နေရာကို ပေးနိုင်ပါတယ်။ မဟုတ်ရင်၊ မေးခွန်းကို အာသော်ရီတီ နာမည် ဆာဗာတွေဆီ ပို့ပေးပါလိမ့်မယ်။

### Authoritative DNS Servers အာသော်ရီတီ DNS ဆာဗာများ

ဒီဆာဗာတွေက ကလိုင်းယင့်က ရယူဖို့ ကြိုးစားနေတဲ့ သတ်မှတ်ထားတဲ့ သတင်းအချက်အလက်ကို သိမ်းဆည်းထားပါတယ်။ ဒါကြောင့် ၎င်းတို့က ရီကာဆစ် DNS ဆာဗာအတွက် လိုအပ်တဲ့ ဝက်ဘ်ဆိုက် နာမည်-ကို-IP-လိပ်စာ ဘာသာပြန်မှုကို ပေးပါတယ်။ ဒီဆာဗာတွေဟာ ကုမ္ပဏီတွေက ၎င်းတို့ရဲ့ နာမည်တွေ – **ဒိုမိန်းများ** (domains) – ဥပမာ Microsoft.com ဒါမှမဟုတ် samsscoops.com လို မှတ်ပုံတင်တဲ့ နေရာတွေပါ။ ဒီဆာဗာတွေကို နေရာ ရာနဲ့ချီပြီး ဖြန့်ကျက်ထားပြီး၊ အင်တာနက်ကို ပံ့ပိုးဖို့ လိုအပ်တဲ့ DNS ဆာဗာ အများအပြားကို ပေးပါတယ်။

ဆာဗာ အမျိုးအစား လေးမျိုးရဲ့ ဖော်ပြချက်တွေကနေ သတိပြုနိုင်သလို၊ နာမည်-ကို-IP-လိပ်စာ ဘာသာပြန်မှု တောင်းဆိုတာက မေးခွန်းကို ဖြေပြီးတဲ့အထိ ဆာဗာ အဆင့် တစ်ခုစီနဲ့ ထပ်ခါထပ်ခါ ဖြစ်ပါတယ်။ အဖြေက ရီကာဆစ် DNS ဆာဗာမှာ ရှိရင် နောက်ထပ် အဆင့်တွေ မလိုအပ်ပါဘူး။ ဒါပေမယ့် မရှိရင်၊ အဖြေကို ပြန်ရသည်အထိ ဆာဗာ အမျိုးအစား တစ်ခုစီကနေ ဖြတ်ပြီး ပို့ရပါတယ်။

အခု၊ DNS အားနည်းချက်တွေနဲ့ ၎င်းတို့နဲ့အတူ ပါလာတဲ့ တိုက်ခိုက်မှုတွေအကြောင်း ပိုမို လေ့လာဖို့ ဆက်သွားကြပါစို့။

## DNS Attacks DNS တိုက်ခိုက်မှုများ

DNS ရဲ့ ဖွဲ့စည်းပုံဟာ ၎င်းရဲ့ အင်အားအကောင်းဆုံးနဲ့ အားနည်းဆုံး ဂုဏ်ရည်နှစ်ခုစလုံးပါ။ တစ်နည်းအားဖြင့်၊ ဆာဗာ အများအပြားနဲ့ဆိုရင် ၎င်းဟာ အလွန် မြန်ဆန်ပြီး ယုံကြည်စိတ်ချရပြီး၊ ပြဿနာတစ်ခု ရှိရင် ဆာဗာတွေက တစ်ခုကနေ တစ်ခုကို လွှဲယူနိုင်ပါတယ်။ ဒါပေမယ့် ဆာဗာ အများအပြား ရှိတာက တိုက်ခိုက်မှု အခွင့်အလမ်းတွေ ပိုများလာစေပါတယ်။ ဘာကြောင့်လဲဆိုတော့ အားလုံးကို တပြိုင်နက် ကာကွယ်တာဟာ မဖြစ်နိုင်တဲ့ အလုပ်တစ်ခုနီးပါးဖြစ်ပြီး၊ ၎င်းတို့ကို မတူညီတဲ့ အဖွဲ့တွေနဲ့ ကုမ္ပဏီတွေက စီမံခန့်ခွဲထားလို့ပါ။ DNS ဟာ ၎င်းရဲ့ ဖွဲ့စည်းပုံတစ်လျှောက် မတူညီတဲ့ တိုက်ခိုက်မှု အမျိုးအစား အများအပြားကို ခံနိုင်ရည်ရှိပြီး၊ ဒီမှာ အဓိက တိုက်ခိုက်မှု အမျိုးအစား ငါးမျိုးကို ဖော်ပြထားပါတယ်။

### DNS Spoofing DNS အတုလုပ်ခြင်း

ဒီတိုက်ခိုက်မှု အမျိုးအစားမှာ DNS ဖြေရှင်းသူတွေမှာ ဝက်ဘ်ဆိုက် နာမည်တွေနဲ့ IP လိပ်စာတွေအကြား ဘာသာပြန်မှုတွေ သိမ်းဆည်းထားတဲ့ DNS မှတ်တမ်းတွေကို ပြောင်းလဲတာ၊ ၎င်းတို့ရဲ့ ကက်ရှ် (cache) ဒါမှမဟုတ် မှတ်ဉာဏ်ကို ပြင်ဆင်တာ ပါဝင်ပါတယ်။ ဒါက တစ်ယောက်ယောက်ကို ဗိုင်းရပ်စ် ပို့ဆောင်တဲ့ ဆိုက်တစ်ခု ဒါမှမဟုတ် အခြား ဆိုက်တစ်ခုကို အတုလုပ် ဒါမှမဟုတ် တုပထားတဲ့ ဆိုက်ဆီ လမ်းကြောင်းပြောင်းပို့တာကို ဆိုလိုပါတယ်။ ဒီတိုက်ခိုက်မှုတွေဟာ အန္တရာယ်ရှိပါတယ်။ ဘာကြောင့်လဲဆိုတော့ သင်ဟာ ဒီအတု ဆိုက်တွေမှာ စကားဝှက်လို အရေးကြီး သတင်းအချက်အလက်တွေကို ထည့်သွင်းမိနိုင်ပြီး၊ အဲဒါကို တိုက်ခိုက်သူဆီ ပေးအပ်မိနိုင်လို့ပါ။

### DNS Hijacking DNS ချိတ်ဆွဲခြင်း

ချိတ်ဆွဲခြင်း (Hijacking) က အတုလုပ်ခြင်း (spoofing) နဲ့ ဆင်တူပေမယ့်၊ DNS မေးခွန်းတွေကို တိုက်ခိုက်သူရဲ့ ထိန်းချုပ်မှု အောက်က DNS ဆာဗာဆီ လမ်းကြောင်းပြောင်းပို့တာပါ၊ အောက်က ပုံမှာ ဖော်ပြထားတဲ့အတိုင်း။ ဒါက သူတို့က အသုံးပြုသူတွေကို သူတို့ လိုချင်တဲ့ ဘယ်နေရာကိုမဆို ပို့နိုင်တယ်ဆိုတာကို ဆိုလိုပါတယ်။ ဥပမာ၊ မူလအတိုင်း အတိအကျ တူတဲ့ အတု ဘဏ်ဆိုက်တစ်ခုမှာ တစ်ယောက်ယောက်က ၎င်းတို့ရဲ့ အသေးစိတ် အချက်အလက်တွေ ရိုက်ထည့်လိုက်ရင်၊ တိုက်ခိုက်သူက အဲဒါကို အသုံး၍ ၎င်းတို့ရဲ့ ဘဏ်အကောင့်ထဲ ဝင်ရောက်နိုင်ပါတယ်။

### DNS Tunneling DNS ဥမင်လိုဏ်ခေါင်းဖောက်ခြင်း

ဒီတိုက်ခိုက်မှုက DNS မေးခွန်းတွေနဲ့ တုံ့ပြန်မှုတွေရဲ့ ပေးဝန်ဒေတာ (payload data) ထဲမှာ အခြား ပရိုတိုကောတွေကို ဖုံးကွယ်တာပါ။ တိုက်ခိုက်သူတွေက ဒါကို ဆာဗာတစ်ခုကို ဝေးကွာစွာ ထိန်းချုပ်ဖို့ မာလ်ဝဲး (malware) ဒါမှမဟုတ် အခြား ပရိုတိုကောတွေ ဖြတ်ပို့ဖို့ အသုံးပြုပါတယ်။ ဥမင်လိုဏ်ခေါင်း တိုက်ခိုက်မှု (Tunneling attacks) တွေမှာ များသောအားဖြင့် ဖိုင်ယာဝေါ (firewall) ကနေ တစ်ခုခုကို ဖုံးကွယ်တာပါဝင်ပြီး၊ အပြင်မှာ ဖိုင်ယာဝေါက မြင်တဲ့ ပရိုတိုကော တစ်မျိုးရှိပေမယ့်၊ အထဲမှာ အမှန်တကယ် အလွန်ကွဲပြားတာပါ။ ဒါက ပစ္စည်းတစ်ခုကို ၎င်းရဲ့ မူလ ထုပ်ပိုးမှုနဲ့ ယူပြီး၊ အဲဒီ ထုပ်ပိုးမှုထဲမှာ တစ်ခြားအရာတစ်ခုနဲ့ အစားထိုးကာ၊ ပြီးတော့ မူလအတိုင်း လို ဟန်ဆောင်ပြီး ဖြတ်ပို့တာနဲ့ တူပါတယ်။

### DNS Amplification DNS ချဲ့ထွင်တိုက်ခိုက်မှု

ဒီတိုက်ခိုက်မှု အမျိုးအစားက ဝန်ဆောင်မှု ငြင်းပယ်ခြင်း (denial of service) ဖန်တီးဖို့ ရည်ရွယ်ပါတယ်။ တစ်နည်းအားဖြင့်၊ ၎င်းရဲ့ ရည်မှန်းချက်က ဆာဗာတစ်ခုကို ၎င်းရဲ့ ရည်ရွယ်ထားတဲ့ အလုပ်ကို လုပ်ဆောင်နိုင်မှာ မဟုတ်အောင် အလုပ်ပိစေဖို့ပါ။ DNS အတွက်၊ ဒါက အတု အရင်းအမြစ် IP လိပ်စာနဲ့ DNS တောင်းဆိုမှုတွေ ပို့တာပါဝင်ပြီး၊ ဒါက DNS တုံ့ပြန်မှုတွေကို တိုက်ခိုက်သူက ရွေးထားတဲ့ ပစ်မှတ်ဆီ သွားစေပါတယ်။ တိုက်ခိုက်သူတစ်ယောက်က တောင်းဆိုမှု ရာနဲ့ချီဖန်တီးပြီး ပစ်မှတ်ကို အလုပ်ပိစေနိုင်ပါတယ်။ ဒီတိုက်ခိုက်မှုက အများသုံး ရရှိနိုင်တဲ့ DNS ဆာဗာတွေကို အသုံးပြုပါတယ်။

### DNS Flood Attack DNS ရေလွှမ်းတိုက်ခိုက်မှု

DNS ရေလွှမ်းတိုက်ခိုက်မှု (DNS flooding) က ချဲ့ထွင်တိုက်ခိုက်မှု (amplification) နဲ့ ဆင်တူပေမယ့်၊ အခြား အရင်းအမြစ် ဆာဗာကို ပစ်မှတ်ထားတာမဟုတ်ဘဲ၊ အတု DNS တောင်းဆိုမှု ထောင်နဲ့ချီပို့ပြီး DNS ဆာဗာ ကိုယ်တိုင်ကို အလုပ်ပိစေတဲ့ တိုက်ခိုက်မှုပါ။ ဒါက စစ်မှန်တဲ့ တောင်းဆိုမှု တစ်ခုခုကို အလုပ်မလုပ်နိုင်အောင် တားဆီးပါတယ်။

## Conclusion နိဂုံး

DNS ဟာ ကမ္ဘာတဝှမ်းကနေ တစ်စက္ကန့်ကို တောင်းဆိုမှု သိန်းနဲ့ချီတဲ့ အရေအတွက်ကို ကိုင်တွယ်ဖို့ မတူညီတဲ့ DNS ဆာဗာ အမျိုးအစား လေးခုရဲ့ ဖွဲ့စည်းပုံကို အားကိုးပါတယ်။ သင့်ကွန်ပျူတာက ပထမဆုံး အသုံးပြုတဲ့ ဆာဗာက ရီကာဆစ် ဆာဗာပါ။ ဒါက သင့်စက်က တိုက်ရိုက် ဆက်သွယ်တဲ့ ဆာဗာပါ။ အကယ်၍ ရီကာဆစ် ဆာဗာက တောင်းဆိုမှုရဲ့ အဖြေကို မသိရင်၊ ၎င်းက လိုအပ်တဲ့ DNS တုံ့ပြန်မှု ရှာတွေ့သည်အထိ အခြား ဆာဗာ အမျိုးအစား သုံးခုကို ဖြတ်ပြီး ပို့ပေးပါတယ်။ ဒီဖွဲ့စည်းပုံရဲ့ အင်အားက ၎င်းရဲ့ ခံနိုင်ရည်ရှိမှုနဲ့ ရရှိနိုင်မှုပါ။ ၎င်းရဲ့ အားနည်းချက်ကတော့ တိုက်ခိုက်သူတွေအတွက် အဓိက ပစ်မှတ်ဖြစ်တာပါ။ ဘာကြောင့်လဲဆိုတော့ ၎င်းတို့က အသုံးပြုသူ အများအပြားကို တပြိုင်နက် ပစ်မှတ်ထား တိုက်ခိုက်နိုင်ပြီး၊ မတူညီတဲ့ တိုက်ခိုက်မှု အမျိုးအစားတွေ အသုံးပြုလို့ပါ။ DNS ဆာဗာတွေနဲ့ ၎င်းတို့ကို ဆန့်ကျင်တဲ့ တိုက်ခိုက်မှုတွေအကြောင်း လေ့လာတာက အလွန် အရေးကြီးပါတယ်။
